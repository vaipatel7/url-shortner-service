Scenario: Creating a Short URL and Resolving Redirect

Given
A running URL shortener service with a REST API.
The database is empty.
A user provides a valid long URL (https://example.com) and optional custom alias (abc123).

When
The user calls the POST /urls endpoint to create a short URL.
The user then calls the GET /:alias endpoint to resolve the short URL.

Then
The service responds with HTTP 201 and returns the short URL and alias.
The database contains a new record mapping the alias to the long URL.
Resolving the short URL (GET /:alias) returns an HTTP 302 redirect to the original long URL.
A click event is recorded in the database with the correct alias, timestamp, and client metadata.
